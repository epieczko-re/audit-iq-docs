== 25. Model Deployment: Mistral Self-Hosted (Guidance)

____
*v0.2 Addition.* AuditIQ can host Mistral models in-house for LLM-based extraction via the Docling-Graph LiteLLM interface. Self-hosting reduces per-call cost, improves latency and data residency, and aligns with regulated deployments where customer data must not leave the institution's network. This section provides deployment guidance — not binding requirements — for infrastructure teams evaluating self-hosted LLM inference.
____

=== 25.1 Runtime Options

[width="100%",cols="20%,30%,50%",options="header"]
|===
|Runtime |Use Case |Notes

|*vLLM*
|Production GPU inference — high throughput, tensor parallelism, continuous batching
|Recommended for production. Supports multi-GPU sharding, OpenAI-compatible API server, PagedAttention for efficient memory use.

|*Ollama*
|Single-host or developer inference — simple setup, good for testing and private deployment
|Good for Phase 0 POC and local development. Supports quantized models for reduced GPU requirements.

|*LiteLLM*
|Docling-Graph's unified LLM interface — routes to vLLM, Ollama, or cloud providers via a single API
|Docling-Graph integrates natively with LiteLLM. Set `provider_override: mistral` in Docling-Graph config to target self-hosted Mistral.
|===

=== 25.2 Hardware Guidance (Planning Estimates)

____
*Disclaimer:* Hardware requirements depend on the specific Mistral model, quantization level, batch size, and throughput targets. The figures below are planning estimates — validate with infrastructure benchmarks before procurement.
____

[width="100%",cols="25%,25%,25%,25%",options="header"]
|===
|Tier |Model Size |GPU Requirement |Throughput (est.)

|*Development / POC*
|Mistral 7B (quantized)
|1× A10G (24GB) or equivalent
|~50–100 tokens/s per request

|*Staging / small production*
|Mistral Medium (~48B params)
|1× A100 80GB or 2× A100 40GB
|~20–40 tokens/s per request; supports batching

|*Production (high throughput)*
|Mistral Large or custom fine-tune
|2–4× A100 80GB or H100 nodes with tensor parallelism
|Batch processing; target ≥500 pages/hour sustained
|===

=== 25.3 Integration Pattern

[arabic]
. Use *vLLM* (production) or *Ollama* (dev/POC) as the inference runtime.
. Configure Docling-Graph with `provider_override: mistral` and `model_override: <model-name>` via LiteLLM.
. Run inference inside an *internal VPC* with TLS encryption and IAM access controls.
. Expose inference via an *internal inference service* with rate limiting, request batching, health checks, and circuit breakers.
. Instrument per-call GPU seconds and estimated cost for chargeback and monitoring.

*Example Docling-Graph config (YAML):*

[source,yaml]
----
pipeline:
  backend: llm
  inference: remote
  provider_override: mistral
  model_override: mistral-medium-latest
  structured_output: true
  use_chunking: true
  litellm_config:
    api_base: "https://llm-internal.auditiq.local:8443/v1"
    timeout: 30
    max_retries: 2
----

=== 25.4 Cost & Operational Tradeoffs

[width="100%",cols="30%,35%,35%",options="header"]
|===
|Factor |Self-Hosted Mistral |Cloud LLM (OpenAI / Anthropic)

|*Per-call cost*
|Low (GPU amortized over volume)
|$0.01–$0.05 per extraction call

|*Infrastructure cost*
|GPU instances ($2–$8/hr per A100); requires SRE/ops
|Zero infra overhead

|*Data residency*
|Full control — data never leaves VPC
|Data sent to third-party; contract required (Section 22.8)

|*Latency*
|Low (local network); ~50–200ms per call
|Variable (100–2000ms); network dependent

|*Scalability*
|Manual — add GPU nodes; implement autoscaling
|Automatic — cloud provider handles scaling

|*Auditability*
|Full control over model weights, versions, logs
|Depends on provider contract for audit access
|===

*Recommendation:* Self-hosted is attractive for AuditIQ because audit workloads are predictable (batch-oriented, not real-time), data is regulatory-sensitive, and per-call volumes at scale make cloud LLM fees significant. Start with cloud LLM for Phase 0 POC, transition to self-hosted for Phase 1a production.

=== 25.5 Security & Licensing

* *VPC isolation:* Run all LLM inference within a private VPC. No public endpoints for inference services.
* *TLS everywhere:* Encrypt all traffic between Docling-Graph, LiteLLM, and the inference runtime.
* *No-training clause:* If using managed Mistral models (Mistral API), ensure contractual ``no training on customer data'' clause per Section 22.8.
* *Model license:* *CRITICAL — obtain legal sign-off* confirming the chosen Mistral model license permits self-hosting under your deployment terms before Phase 1a. Some Mistral models have restrictive licenses for commercial use.
* *PII controls:* Self-hosted inference does not exempt AuditIQ from PII redaction requirements (Section 22.2). Apply the same pre-call PII redaction regardless of inference location.
