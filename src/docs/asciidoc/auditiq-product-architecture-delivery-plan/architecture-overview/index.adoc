== Architecture Overview

=== Architectural Principles

[arabic]
. *Separation of concerns:* Ingestion → Processing → AI → Storage → Reporting.
. *Immutable audit trail:* Every state change is recorded as an event; events are append-only and hash-chained.
. *Declarative configuration:* Packs are JSON, not code.
The platform interprets them.
. *Confidence scoring:* Every output carries a quantified confidence score (extraction, mapping, rule application).
. *Human-in-loop by default:* AI outputs are advisory and non-finalized until explicitly approved by a human with the right role.
. *Multi-tenancy:* Row-level security at the database level.
No cross-tenant data leakage.
See diagram below.

==== Phase 1 Deployment (East US 2)

*Hosting model:* RiskExec-hosted multi-tenant SaaS.
All infrastructure runs in RiskExec's Azure subscription.
Tenant isolation is enforced via PostgreSQL Row-Level Security (RLS), per-tenant Blob containers, and per-tenant Elasticsearch indices.

*Compute:*

* *AKS cluster:* Azure Kubernetes Service (Standard tier).
3 node pools:
** *System pool:* 3× Standard_D4s_v5 (4 vCPU, 16 GB) — core services, orchestration, auth.
** *Pipeline pool:* 3–6× Standard_D8s_v5 (8 vCPU, 32 GB) — document processing, schema mapping, comparison engine.
Autoscales based on queue depth.
** *Infrastructure pool:* 3× Standard_D4s_v5 — Elasticsearch, Qdrant, Redis.

*Database:*

* *Azure Database for PostgreSQL Flexible Server:* General Purpose tier, 8 vCPU, 64 GB RAM.
* Multi-AZ deployment with automatic failover.
* Automated backups: daily full, continuous WAL archiving.
Point-in-time recovery up to 35 days.
* Read replicas: 1 replica for dashboard queries and search indexing.

*Networking:*

* All services deployed within a single VNet.
* AKS, PostgreSQL, and Blob Storage communicate over Private Endpoints (no public internet).
* Azure API Management is the only publicly accessible endpoint.
* Azure Front Door provides WAF, DDoS protection, and TLS termination.
* Network Security Groups (NSGs) restrict traffic between subnets.

*Region:* Azure East US 2 (Virginia).
US data residency guaranteed — no data leaves US Azure regions.

==== Security Architecture

*Authentication & Identity:*

* Azure Active Directory (AAD) as the identity provider.
* OAuth 2.0 / OIDC for all user authentication.
* Service-to-service: Managed Identity (no stored credentials).
* MFA enforced for all human users via AAD Conditional Access.

*Encryption:*

[cols="30%,30%,40%",options="header"]
|===
|Scope |Method |Key Management

|Data in transit
|TLS 1.2+ (enforced; TLS 1.0/1.1 disabled)
|Azure-managed certificates via Front Door

|Data at rest (database)
|AES-256, Transparent Data Encryption (TDE)
|Customer-Managed Keys (CMK) in Azure Key Vault

|Data at rest (blob storage)
|AES-256, Storage Service Encryption (SSE)
|Customer-Managed Keys (CMK) in Azure Key Vault

|Data at rest (search index)
|AES-256
|Azure-managed keys (Phase 1); CMK (Phase 2)

|Secrets & credentials
|Azure Key Vault
|RBAC-scoped access; auto-rotation for DB credentials
|===

*Network security:*

* All data-plane traffic stays within the VNet via Private Endpoints.
* No database or storage account has a public IP.
* Azure Front Door WAF protects against OWASP Top 10 (SQL injection, XSS, etc.).
* DDoS Protection Standard enabled on the VNet.
* NSG rules: deny-all default; explicit allow for service-to-service traffic.

*Tenant isolation:*

* *Database:* PostgreSQL Row-Level Security (RLS) policies on every table.
Every query is scoped to `current_setting('app.tenant_id')`.
* *Blob Storage:* Per-tenant containers with SAS token scoping.
* *Search:* Per-tenant Elasticsearch indices.
* *API:* Tenant ID extracted from JWT and injected into every backend request by the API Gateway.

*Compliance targets:*

* *SOC 2 Type II:* Target readiness by June 30, 2026.
* *PCI DSS:* Not applicable (no payment card data processed).
* *GLBA / Regulation P:* Customer PII handled with encryption and access controls per GLBA requirements.

==== Disaster Recovery

*Strategy:* Multi-AZ within East US 2 (primary), with async replication to West US 2 (secondary) for PostgreSQL and Blob Storage.

*Failover procedure:*

[arabic]
. Azure Monitor detects primary region failure (automated alert).
. PostgreSQL Flexible Server automatic failover to standby replica (< 60 seconds).
. AKS pods auto-restart on healthy nodes within the AZ (< 5 minutes).
. If full-region failure: manual failover to West US 2 secondary.
DNS update via Azure Traffic Manager (RTO: < 1 hour).

==== Scaling Considerations

* *Stateless microservices* (Ingestion, Mapping, Comparison, Reporting): Horizontal pod autoscaler (HPA) based on CPU and custom metrics (queue depth, request latency).
* *Stateful services* (Workflow, Orchestration): Single active instance per tenant with leader election.
Phase 2: partitioned by tenant group for scale.
* *Database:* Vertical scaling (Phase 1).
Read replicas for dashboard queries.
Connection pooling via PgBouncer.
Phase 2: consider Citus for horizontal sharding if > 100 tenants.
* *Document Processing:* Most compute-intensive pipeline.
Dedicated node pool with autoscaling (3–6 nodes).
GPU nodes optional for VLM inference (Phase 2).
* *Document Storage:* Azure Blob auto-scales.
No configuration needed.
* *Search:* Elasticsearch auto-sharding.
ILM policies manage index lifecycle.
