==== AI-Assisted Development

The team will use AI coding assistants (Claude Code, Codex, and similar tools) as a core part of the development workflow.
These tools act as a force multiplier for code generation, boilerplate, test writing, and documentation — effectively doubling or tripling individual developer throughput on implementation tasks.

AI-assisted development does *not* accelerate:

* Architecture and design decisions
* Security reviews and penetration testing
* Compliance and regulatory review
* Customer-facing onboarding and training
* Integration testing with real-world data

Velocity estimates in this plan assume AI-assisted development is in active use from day one.

==== Dependency Chain (Critical Path)

The longest dependency chain runs through Track A:

[arabic]
. Docling-Graph POC (Phase 0, Week 1–3) — *must validate before committing to production pipeline*
. Document processing pipeline (Phase 1a, Week 4–7) — depends on validated extraction
. Schema mapping + confidence scoring (Phase 1b, Week 8–11) — depends on extraction output + pack rules
. Performance tuning + security review (Phase 1c, Week 12–16) — depends on working system

If any item on this chain slips, the launch date is at risk.

==== Parallel Workstreams

* *Infrastructure + auth* (Phase 0) runs parallel to extraction POC
* *UI implementation* (Junior Dev) runs continuously from Week 2 onward, building against API contracts
* *Pack definitions* (Analyst) run continuously from Week 1, providing test data for both tracks
* *Model registry* (Phase 1a) is independent of extraction pipeline and can proceed in parallel
* *Compliance + onboarding work* (Phase 1c) runs parallel to engineering hardening

[[sect-delivery-go-no-go]]
=== Go/No-Go Gate

There is one formal go/no-go gate in the MVP delivery: the *Phase 0 Exit Gate*.
All other quality and risk criteria are tracked in the risk register (<<sect-risks-constraints>>).

==== Phase 0 Exit Gate (March 28, 2026)

*Purpose:* Validate that the Docling-Graph extraction strategy (Decision D1, <<sect-decision-d1>>) produces acceptable results on real HMDA documents before committing 13 weeks of development to building on it.

*Go criteria:*

* Docling-Graph + AFC v2 achieves ≥85% precision on critical HMDA fields (borrower name, loan amount, property address, action taken, loan type) at AFC ≥0.85
* Deterministic reproducibility: same input → same output across runs
* Bounding-box alignment ≥90% overlap with ground truth
* LLM API integration functional via LiteLLM with latency <2s per extraction call

*Kill criteria:*

* <70% precision at AFC ≥0.85 on critical fields
* Cloud OCR fallback required for >40% of pages
* Non-deterministic output across runs

*If killed:* Pivot to Azure AI Document Intelligence as primary extraction with Docling relegated to pre-processing.
This pivot adds ~2 weeks to Phase 1a timeline.
See Decision D1 (<<sect-decision-d1>>) for full pivot rationale.

[[sect-delivery-launch-readiness]]
==== Launch Readiness Checklist (June 2026)

Launch readiness is assessed continuously during Phase 1c rather than at a single gate.
The following must all be true before production launch:

*Engineering:*

* All P0 functional requirements (<<sect-product-requirements>>) pass end-to-end testing
* Extraction latency <2s/page (P95)
* Zero critical or high-severity security findings from penetration test
* Audit trail hash chain verifiable end-to-end (round-trip test)
* Model registry contains approved entries for all production AI components

*Compliance:*

* Third-party AI vendor agreements include exam access and no-training clauses (<<sect-ai-governance>>)
* SOC 2 readiness review complete with no blocking gaps
* PII redaction validated — zero PII leakage to external LLM providers
* All AI-generated content carries required disclaimers and provenance metadata

*Customer:*

* At least 3 beta customers have completed UAT on critical workflows
* Training materials and user documentation available
* Known limitations documented and communicated to beta customers

*Risk:*

* No risk in <<sect-risks-constraints>> with score ≥15 has an unmitigated status
* Override rate on CD-supported schema mappings ≤25%
