=== Overview

An audit pack is a versioned, declarative configuration artifact that encapsulates everything needed to audit a specific regulatory domain or business process. The platform interprets the pack—it does not execute code embedded in the pack. This ensures safety, reproducibility, and extensibility.

*Pack components:*

* Manifest (metadata, versioning, authorship)
* Target schema (which fields are audit-reportable)
* Source schema(s) (document types and expected layouts)
* Comparison rules (how to validate target vs. source)
* Scoring rules (severity, confidence thresholds)
* Report templates (how to format findings into reports)
* Test cases (reference data for validation harness)

*DECISION LOCKED:* Declarative JSON v1 — no executable logic. See Section 19.3.

=== Pack Versioning and Distribution

*Version format:* Semantic Versioning (MAJOR.MINOR.PATCH) - MAJOR: Breaking changes to schemas or rule logic (requires migration) - MINOR: New rules or schemas added (backward-compatible) - PATCH: Bug fixes, typo corrections (no functional changes)

*Pack lifecycle:*
1. *Draft:* Created by pack author, not available to audit runs.
2. *Published:* Available for new audit runs. Existing runs remain pinned to their original version.
3. *Deprecated:* Marked for removal with migration path. No new runs can be created. Existing runs continue.
4. *Retired:* No longer available. Existing runs can reference archived pack version (read-only).

*Distribution channel:*

* *Phase 1:* RiskExec ships HMDA pack v1.0 in the platform binary.
* *Phase 2+:* Packs distributed via pack marketplace API with version pinning and rollback capability.

=== Pack Testing and Validation Harness

*Harness requirements:*

* Load a pack manifest and test cases.
* Execute each test case against the pack rules.
* Compare actual findings to expected findings.
* Report pass/fail, coverage metrics, and rule execution time.

*Test harness success criteria (before publication):*

* All test cases pass.
* Code coverage ≥90% of pack rules executed.
* No test case execution exceeds 5 seconds.
* All AI suggestions (if any) are labeled and traced to model/version.

=== Pack Interface Contract (System-to-Pack)

The platform interacts with packs via the following immutable contract:

[source,typescript]
----
interface AuditPackContract {
  // Manifest loading
  loadManifest(packId: string, version: string): Promise<PackManifest>;

  // Schema validation
  validateTargetRecord(record: object, packId: string): ValidationResult;
  validateSourceDocument(doc: object, docType: string, packId: string): ValidationResult;

  // Mapping suggestion
  suggestMappings(
    targetSchema: object,
    sourceSchema: object,
    packId: string
  ): MappingSuggestion[];

  // Comparison execution (deterministic)
  executeComparisons(
    targetRecord: object,
    sourceData: object,
    mappings: FieldMapping[],
    packId: string,
    packVersion: string
  ): ComparisonResult[];

  // Scoring
  scoreFindings(findings: Finding[], packId: string): ScoredFinding[];

  // Report generation
  generateReport(auditData: AuditData, packId: string): Report;
}
----

*Immutability guarantee:* For a given (packId, packVersion, targetRecord, sourceData, mappings), the output of executeComparisons() is always identical. All non-deterministic outputs (AI narratives) are explicitly generated separately and labeled.
