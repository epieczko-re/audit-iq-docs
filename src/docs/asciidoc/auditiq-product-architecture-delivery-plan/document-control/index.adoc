=== How to Use This Document

This document serves three audiences simultaneously:

* *Engineering team:* Sprint-plan against Sections 7–12 (Key Architectural Concepts through Data Model), Sections 20–23 (MVP Decisions, Immutability, Exam Bundle, AI Governance), and Sections 24–27 (UI Workflows, Model Registry, Model Deployment, Docling-Graph Integration). Each section contains explicit contracts, schemas, and acceptance criteria that can be decomposed into user stories and tasks. See the companion WBS (AuditIQ_WBS_v0.2.xlsx) for task breakdown.
* *Product / leadership:* Read Sections 1–6 for context and competitive positioning. Review <<sect-pricing, Pricing and Packaging>>, the Delivery Plan, <<sect-mvp-decisions, MVP Decisions Locked>>, and the MVP Cutline (AuditIQ_MVP_Cutline_v0.2.md) for investment decisions. The Risks, Constraints, and Mitigations section and <<sect-decisions, Decisions and Open Questions>> provide risk and decision context.
* *Compliance / regulatory stakeholders:* Focus on <<sect-scoring-framework, Scoring Framework>>, <<sect-immutability, Immutability vs. Correctability>>, <<sect-exam-ready-bundle, Exam-Ready Bundle>>, and <<sect-ai-governance, AI Governance & Model Risk Controls>> for regulatory defensibility. <<sect-model-registry, Model Registry>> and the Glossary (<<sect-appendices, Appendices>>) provide supporting reference material.

*Canonical Glossary:* All confidence metrics (AFC, mapping_confidence, extraction_confidence, rule_confidence, combined_confidence), routing thresholds, severity definitions, and calibration criteria are defined in a *single source of truth*: the *Canonical Glossary* (Appendix — Glossary). If a threshold value elsewhere in this document conflicts with the Glossary, the Glossary governs.

*Conventions used in this document:*

* *TODO* = work item that must be completed before implementation starts
* *ASSUMPTION* = a belief stated explicitly; risk if wrong is documented
* *DECISION NEEDED* = requires stakeholder input before proceeding
* *DECISION LOCKED* = decided for MVP; revisit criteria documented in <<sect-mvp-decisions>>
* Reference documents are listed in Appendix D.
