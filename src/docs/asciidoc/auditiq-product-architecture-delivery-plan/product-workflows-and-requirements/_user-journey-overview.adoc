==== Workflow 1: Audit Lifecycle (End-to-End)

This is the primary happy-path workflow from login to exam-ready bundle export.
It represents the complete user journey through a single audit.

*Actors:* Auditor (primary), System, AI Engine *Trigger:* Auditor creates a new audit *End State:* Exam-ready bundle exported; all events immutable

[mermaid]
----
flowchart TD
    A([Auditor Logs In]) --> B[Create New Audit<br/>Select Pack Template]
    B --> C[Configure Audit Parameters<br/>Loan Sample, Date Range]
    C --> D[Upload Loan Documents<br/>CD, 1003, Appraisal, Title]
    D --> E[Docling Extraction<br/>+ AFC Scoring]
    E --> F[Schema Mapping<br/>AI Suggestions + Review]
    F --> G[Rules Engine Scoring<br/>Pack Rules × Extracted Data]
    G --> H[AI Generates<br/>Draft Findings]
    H --> I[Reviewer: Effective Challenge<br/>Approve / Edit / Reject]
    I --> J{All Findings<br/>Reviewed?}
    J -->|No — next finding| I
    J -->|Yes| K[Finalize Audit<br/>Lock + Sign Off]
    K --> L[Export Exam-Ready<br/>Bundle — Section 21]
    L --> M([Audit Complete])

    %% Audit Trail annotation (right side)
    N[Audit Trail:<br/>All Events Immutable<br/>RFC 8785 + Hash Chain]
    E -.-> N
    F -.-> N
    G -.-> N
    H -.-> N
    I -.-> N
    K -.-> N
    L -.-> N

This workflow details the document processing pipeline from upload through field-level confidence routing.

*Actors:* Auditor (upload), System (Docling, OCR, AFC), Reviewer (validation) *Trigger:* Auditor uploads one or more loan documents *End State:* All extracted fields confirmed (auto, reviewed, or manual) and ready for schema mapping

*Design Reference:* <<sect-doc-processing>>, <<sect-decision-d1, Decision D1>>

[mermaid]

    D --> E[AFC Score Computation<br/>5 Weighted Signals]

    %% AFC signal weights annotation
    E -.-> AFC[docling_page_grade 0.15<br/>ocr_segment_conf 0.25<br/>pattern_validity 0.25<br/>layout_anchor 0.20<br/>cross_doc_redundancy 0.15]

    E --> F{AFC Score<br/>Routing?}

    L -->|Override| K
    K --> M([Field Value Confirmed<br/>→ Schema Mapping])
    G --> M

This workflow details how extracted fields are mapped to the HMDA/MISMO target schema with AI assistance and human oversight.

*Actors:* System (matcher), AI (calibration), Reviewer (approval) *Trigger:* Extracted fields arrive from Document Processing Service *End State:* All fields mapped to target schema; approved mappings feed into rules engine

*Design Reference:* <<sect-schema-mapping-service>>, <<sect-decision-d2, Decision D2>>

[mermaid]
----
flowchart TD
    A([Extracted Fields<br/>from Docling + AFC]) --> B[Hybrid Heuristic Matcher<br/>5 Weighted Signals]
    B --> C[Layer 1: Raw Signals<br/>conf_extraction, conf_mapping, conf_rule]

    %% Coverage note annotation
    C -.-> COV[Coverage Note:<br/>CD-only = 10–20% at ≥ 0.85<br/>Full coverage requires<br/>1003 + Appraisal + Title]

    C --> D[Layer 2: Calibrated Score<br/>Temperature Scaling /<br/>Isotonic Regression]
    D --> E{Routing<br/>Decision?}
    E -->|≥ 0.95| F([Auto-Accept<br/>calibrated ≥ 0.95])

==== Workflow 4: Findings Review & Approval (AI Governance Controls)

This workflow details the human oversight process for AI-generated findings, enforcing the AI governance red lines from <<sect-ai-governance>>.

*Actors:* Rules Engine, AI (LLM), Reviewer (effective challenge) *Trigger:* Rules engine generates exceptions from scored data *End State:* Finding finalized with full provenance and effective challenge documentation

*RED LINES (<<sect-ai-governance>>):*

* No AI output enters the official audit record without explicit human approval
* AI output must never be represented as definitive conclusions
* Reviewer is responsible for accuracy of any approved finding

*Required Disclaimers (<<sect-ai-governance>>):*

* "This content was drafted by an AI assistant and requires human verification before use"
* "AI-generated content may contain inaccuracies — reviewer is responsible for verifying accuracy"
* "This finding has been reviewed and approved by [reviewer name] on [date]"

*Design Reference:* Findings Management, Reporting Service, <<sect-ai-governance>>, <<sect-decision-d1, Decision D1>>–<<sect-decision-d3, D3>>

[mermaid]

==== Workflow 5: Audit Execution — Manual Mode

Fully manual audit where the reviewer performs all field comparisons without automated extraction.

*Actors:* Admin (setup), Reviewer (comparison + findings), Manager (approval + reporting)
*Trigger:* Admin creates a new audit run in Manual execution mode
*End State:* All records reviewed, findings generated, report exported

*Design Reference:* <<sect-concept-of-operations>>

[mermaid]
----
sequenceDiagram
  participant Admin
  participant System
  participant Reviewer
  participant Manager

  Admin->>System: Create audit run + select pack
  Admin->>System: Upload target dataset (HMDA LAR)
  System->>System: Validate schema (37 Key Data Fields)
  System-->>Admin: Validation result (errors/warnings)
  Admin->>System: Link source documents to LAR records
  Reviewer->>System: Open record in side-by-side viewer
  Reviewer->>System: Stage 1: Compare fields (confirm / discrepancy / unable)
  Reviewer->>System: Create finding (value, root cause, severity, comment)
  Reviewer->>System: Stage 2: Substantive review of flagged records
  System->>System: Route findings to responsible parties
  Reviewer->>System: Track remediation + evidence
  System->>System: Second review (QA sampling)
  Manager->>System: Review dashboards
  Manager->>System: Generate audit summary report
  System-->>Manager: Exam-ready bundle
----

==== Workflow 6: Audit Execution — OCR-Assisted Mode (Phase 1+)

Docling extracts data from source documents and the comparison engine generates preliminary findings, with human review focused on uncertain cases.

*Actors:* Admin (setup), System (Docling, Comparison Engine), Reviewer (validation + findings), Manager (approval + reporting)
*Trigger:* Admin creates a new audit run in OCR-Assisted execution mode
*End State:* All records reviewed (auto-confirmed + human-validated), findings generated, report exported

*Design Reference:* <<sect-concept-of-operations>>

[mermaid]
----
sequenceDiagram
  participant System
  participant Docling
  participant CompEngine as Comparison Engine
  participant Reviewer
  participant Manager

  System->>Docling: Send source documents
  Docling->>System: Extracted text, tables, key-value pairs + AFC scores
  System->>CompEngine: Stage 1: Apply pack rules (target vs. extracted source)
  CompEngine->>System: Preliminary findings + combined confidence scores
  System->>Reviewer: Pre-populated comparison results (tiered by confidence)
  Note over Reviewer: AFC >= 0.85: auto-confirmed (overridable)
  Note over Reviewer: 0.50–0.84: flagged for human review
  Note over Reviewer: < 0.50: manual review required
  Reviewer->>System: Verify flagged items + override if needed
  Reviewer->>System: Stage 2: Substantive review of flagged records
  Reviewer->>System: Create/adjust findings
  System->>Manager: Escalated findings for review
  Manager->>System: Approve / request re-review
  Manager->>System: Generate report + exam-ready bundle
----

'''

==== Workflow 7: Audit Execution — Automated Pre-Scan ("Target-to-Source" / "Record-to-Record")

For structured data audits (e.g., HMDA LAR against LOS database extract), the full Stage 1 comparison can be automated.

*Actors:* Admin (setup + mapping approval), System (Comparison Engine), Reviewer (finding review), Manager (approval + reporting)
*Trigger:* Admin creates a new audit run in Automated Pre-Scan execution mode with structured source data
*End State:* All records compared automatically, human review completed for flagged findings, report exported

*Design Reference:* <<sect-concept-of-operations>>

[mermaid]
----
sequenceDiagram
  participant Admin
  participant System
  participant CompEngine as Comparison Engine
  participant Reviewer
  participant Manager

  Admin->>System: Upload target dataset (CSV/API)
  Admin->>System: Upload source dataset (CSV/API)
  System->>System: Auto-suggest schema mapping + confidence scores
  System-->>Admin: Present mapping suggestions (tiered by confidence)
  Admin->>System: Approve, adjust, or manually map fields
  System->>CompEngine: Run all pack rules automatically
  CompEngine->>System: Full findings register + combined confidence scores
  System->>Reviewer: Critical/High findings for review
  System->>Reviewer: Below-threshold (< 0.85) findings for review
  Reviewer->>System: Approve / override findings
  System->>Manager: Escalated findings
  Manager->>System: Final approval
  Manager->>System: Generate report + exam-ready bundle
