== Product Workflows & Requirements

____
This section is the single reference for every MVP screen, workflow, functional requirement, and architecture detail in AuditIQ.
It is organized by user journey so each execution path shows the workflow, screen walkthrough, requirements, and architecture context together.

*How to read this section:*

* *Journey Overview* — audit lifecycle and execution-mode comparison.
* *Mode-Specific Journeys* — Manual, OCR-Assisted, and Automated Pre-Scan flows with step-by-step screen walkthroughs.
* *Shared Journeys* — finding review, remediation/reporting, dashboards, and administration.
* *Non-Functional Requirements* — cross-cutting quality attributes and performance targets.
* *Reference Material* — global layout, screen inventory, and UI development strategy.

*Read this section first when estimating sprint work.*
Each journey maps directly to one or more sprints in the Workflow Delivery Matrix (<<sect-workflow-delivery-matrix>>).
