== Product Workflows & Requirements

____
This section is the single reference for every MVP screen, workflow, functional requirement, and architecture detail in AuditIQ.

**Organization:** This section is organized around **user journeys**.
Each journey is an end-to-end specification containing workflow diagrams, screens and wireframes, functional requirements (by epic), architecture details (components, data flows, API contracts), NFRs, and acceptance criteria.

*How to read this section:*

* *Journey-to-Screen Map* — connects each journey to execution mode, primary screens, and coverage.
* *Journeys* — one document per user journey containing flow diagrams → screens → requirements → architecture → NFRs → acceptance criteria.
* *Reference Material* — global layout, screen inventory, and UI development strategy.

Read this section first when estimating sprint work.
____
