== Product Workflows & Requirements

____
This section is the single reference for every MVP screen, workflow, functional requirement, and architecture detail in AuditIQ.
It is organized by audit lifecycle stage so that each stage contains the user journey, screen wireframes, functional requirements (epics), and architecture in one place.

*How to read this section:*

* *User Journey Overview* — seven end-to-end workflow diagrams showing the primary user journeys through the system.
* *Screen Inventory* — quick-reference table of all 26 MVP screens with priority and component notes.
* *Stages 1–6* — per-stage detail: user journey diagrams, screen wireframes with interactions, functional requirements organized by epic, and service architecture with sequence diagrams.
* *Non-Functional Requirements* — cross-cutting quality attributes and performance targets.
* *Acceptance Criteria Summary* — summary of acceptance testing approach.

*Read this section first when estimating sprint work.*
Each lifecycle stage maps directly to one or more sprints in the Workflow Delivery Matrix (<<sect-workflow-delivery-matrix>>).
