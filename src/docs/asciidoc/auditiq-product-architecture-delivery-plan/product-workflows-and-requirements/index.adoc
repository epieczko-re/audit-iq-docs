== Product Workflows & Requirements

____
This section is the single reference for every MVP screen, workflow, functional requirement, and architecture detail in AuditIQ.
It is organized around user journeys first, with the supporting stage details immediately after each journey context so readers can move from flow to screens to requirements without jumping sections.

*How to read this section:*

* *User Journey Overview* — seven end-to-end workflow diagrams across the three execution modes (Manual, OCR-Assisted, Automated Pre-Scan).
* *Stages 1–6* — per-stage detail aligned to the journeys: screens, wireframes, interactions, functional requirements by epic, and service architecture.
* *Non-Functional Requirements* — cross-cutting quality attributes and performance targets.
* *Acceptance Criteria Summary* — summary of acceptance testing approach.
* *Reference Material* — global layout, screen inventory, and UI development strategy for implementation lookup.

*Read this section first when estimating sprint work.*
Each lifecycle stage maps directly to one or more sprints in the Workflow Delivery Matrix (<<sect-workflow-delivery-matrix>>).
