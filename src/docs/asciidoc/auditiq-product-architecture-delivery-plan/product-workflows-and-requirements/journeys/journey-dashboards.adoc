=== Journey: Dashboards & Analytics

Dashboards provide read-only operational and executive views across all audits.

==== End-to-End Workflow Diagrams

===== Dashboard Data Aggregation & Rendering

This workflow shows how audit events flow from operational services through aggregation into the executive and manager dashboards, including tenant isolation and role-based filtering.

*Actors:* System (event processing + aggregation), Executive (read-only consumer), Manager (operational consumer) *Trigger:* Any audit event is emitted (finding created/closed, remediation updated, report generated) *End State:* Dashboard KPIs and charts reflect current audit program state within 30 seconds

*Design Reference:* <<sect-reporting-service>>, <<sect-audit-trail-event-store>>

[mermaid]
----
flowchart TD
    A([Audit Events Emitted<br/>Findings, Remediations,<br/>Reports, Approvals]) --> B[Immutable Event Store<br/>Append-Only Log]
    B --> C[Aggregation Engine<br/>Materialized Views + Queries]
    C --> D{User Role?}

    D -->|Executive| E[Executive Dashboard<br/>Portfolio KPIs]
    D -->|Manager| F[Manager Dashboard<br/>Operational Metrics]
    D -->|Model Risk Officer| G[Drift Monitoring<br/>Phase 2]

    E --> E1[KPI Cards:<br/>Audit Count, Findings,<br/>Compliance Rate, Avg Resolution]
    E --> E2[Compliance Trend Chart<br/>12-Month Rolling]

    F --> F1[Active Audits<br/>+ Progress Tracking]

===== Executive & Manager Dashboard Interaction

This sequence diagram shows the interaction flow for both executive and manager dashboard users, including how alerts surface SLA violations and how drill-down navigation connects dashboards to operational screens.

*Actors:* Executive, Manager, System *Trigger:* User navigates to dashboard *End State:* User has visibility into audit program health and has actioned any urgent items

[mermaid]
----
sequenceDiagram
    participant Executive
    participant Manager
    participant System
    participant DB as PostgreSQL (RLS)

    Executive->>System: Open Executive Dashboard
    System->>DB: Query KPIs (filtered by tenant_id via RLS)
    DB-->>System: Aggregated metrics (audit count, findings, compliance rate)
    System-->>Executive: Render KPI cards + compliance trend chart

    Executive->>System: Click compliance trend data point (March)
    System->>DB: Query March audits for tenant
    System-->>Executive: Drill-down: March audit list with findings summary

    Executive->>System: Download Quarterly Report
    System->>System: Generate executive summary PDF
    System-->>Executive: PDF download

    Note over Manager: Manager Dashboard — operational view

    Manager->>System: Open Manager Dashboard
    System->>DB: Query active audits, critical findings, pending approvals (RLS-filtered)
    DB-->>System: Operational data + SLA violation alerts
    System-->>Manager: Render dashboard with alert banner

    alt SLA violations detected
        System-->>Manager: Alert: "2 Critical findings untriaged > SLA"
        Manager->>System: Click [Triage Now] on critical finding
        System-->>Manager: Navigate to Finding Detail (triage pre-expanded)
    end

    Manager->>System: Review pending AI narrative approval
    System-->>Manager: Navigate to AI Narrative Approval screen
    Manager->>System: Approve AI narrative
    System->>System: Record human_approval event

    Manager->>System: Check team workload
    System-->>Manager: Reviewer assignment counts + overdue items
    Manager->>System: Navigate to Aging Dashboard for overdue details
----

'''

* *[Triage Now]* navigates to <<wf-finding-detail>> with triage section pre-expanded
