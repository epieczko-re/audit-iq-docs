==== Happy Path

[arabic]
. Manager creates a new audit → selects HMDA origination pack → defines sample (50 loans) and date range.
. Manager uploads HMDA LAR CSV (target dataset) → system validates 37 Key Data Fields against pack schema → confirms record count.
. Reviewer uploads source documents (CDs, 1003s) → system auto-classifies and matches to LAR records by loan number.
. Audit enters `DocumentsUploaded` state → extraction begins automatically.

==== Key Edge Cases

* *Schema validation failure:* Missing required columns → system rejects upload with specific error listing; user re-uploads corrected file.
* *Unmatched documents:* Document classification confidence < 0.85 → routed to manual classification queue.
* *Oversized file:* > 100 MB single file → client-side rejection before upload begins (FR-1.5).

===== Audit Lifecycle Architecture

*Component:* Audit Lifecycle Manager (<<sect-component-design>>)

*State management:* The Audit Lifecycle Manager owns the state machine and enforces valid transitions.
Each transition emits an immutable audit event to the Event Store.
Invalid transitions (e.g., jumping from `Created` to `Finalized`) are rejected with a descriptive error.

*Progress tracking:* The progress monitor aggregates status from downstream services (Ingestion Service, Document Processing Service, Schema Mapping Service, Comparison Engine, Findings Management Service) to compute per-phase completion percentages.

*Execution mode routing:* The selected execution mode determines which downstream services are invoked and in what order:

* *Manual:* Ingestion → Document Upload → Manual Review (skip automated extraction and comparison)
* *OCR-Assisted:* Ingestion → Document Upload → Docling Extraction + AFC → Schema Mapping → Comparison Engine → Review
* *Automated Pre-Scan:* Ingestion → Schema Mapping → Comparison Engine → Review (both target and source are structured data)

==== Developer Context

*Services:* Ingestion Service, Azure Blob Storage (encryption + quarantine), Document Classifier, Audit Lifecycle Manager.
*Data flow:* CSV → Ingestion Service (validation + SHA-256 checksums) → PostgreSQL staging tables.
PDFs → Blob Storage → Document Classifier → link to LAR records.
*API contracts:* `IngestionRequest`, `ValidationResult` interfaces (see Architecture above).
