=== Top 5 Technical Risks

These risks have the highest impact on the MVP timeline.
Full details in <<sect-risks-constraints>>.

[cols="5%,35%,10%,50%",options="header"]
|===
|ID |Risk |Score |Top 3 Mitigations

|*R4*
|LLM hallucinations in findings narratives mislead reviewers
|*20*
|_(1)_ Require explicit human approval before finalizing any AI narrative.
_(2)_ Citation check pipeline.
_(3)_ Confidence threshold (min 0.70).

|*R24*
|Non-canonical JSON serialization breaks hash chain verification
|*20*
|_(1)_ Mandate RFC 8785 (JCS) for all hashed payloads.
_(2)_ Round-trip verification test on every write.
_(3)_ Pin serialization library versions.

|*R26*
|Automation bias: reviewers rubber-stamp AI outputs
|*20*
|_(1)_ Mandatory effective challenge documentation.
_(2)_ Randomized challenge prompts.
_(3)_ Track override rates per reviewer.

|*R2*
|Schema matcher confidence scores poorly calibrated (>40% override rate)
|*16*
|_(1)_ Expand training data.
_(2)_ Adjust confidence thresholds.
_(3)_ Conservative initial thresholds (>=0.95 auto-accept).

|*R7*
|Q2 timeline slip due to Docling/OCR integration delays
|*16*
|_(1)_ Start Docling setup in Feb (now).
_(2)_ Parallel track: manual extraction UI ready by week 2.
_(3)_ Reduce scope to core workflow only.
|===

NOTE: R4, R24, and R26 are the *top 3 risks* — all score 20 (Likelihood x Impact).
R4 and R26 both relate to AI trust; R24 relates to audit trail integrity.
These three risks should be reviewed weekly during standup.
